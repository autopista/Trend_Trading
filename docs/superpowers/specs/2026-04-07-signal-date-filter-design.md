# 시그널 날짜별 조회 기능 설계

## 배경

현재 시그널 목록 카드는 최근 7일치 시그널을 confidence 내림차순으로 전부 보여준다.
같은 종목이 매일 시그널을 생성하므로 동일 종목이 중복 표시되어 가독성이 떨어진다.

## 목표

- 날짜별로 시그널을 조회하여 중복 제거
- KR/US 각 마켓 대시보드에서 독립적으로 날짜 선택
- 캘린더 팝업으로 시그널이 존재하는 ��짜를 시각적으로 표시

## 요구사항

1. 시그널 목록 헤더에 날짜 네비게이션 (◀ 날짜 ▶) 추���
2. 날짜 클릭 시 캘린더 팝업 표시
3. 캘린더에서 시그널이 존재하는 날짜에 파란 점 표시, 없는 날짜는 클릭 불가
4. ◀ ▶ 버튼은 시그널이 있는 이전/다음 날짜로 점프
5. buy/sell/watch 전부 표시하되 기존 타입별 필터 유지
6. Summary 카드(매수/매도/관찰 카운트)는 선택된 날짜 기준으로 연동
7. 마켓 전환 시 각각의 마지막 선택 날짜를 localStorage로 기억
8. 초기 로딩 시 localStorage에 저장된 날짜 사용, 없으면 가장 최근 시그널 날짜

## 접근법

**API 파���미터 방식** — 기존 엔드포인트에 쿼리 파라미터를 추가하고, 캘린더용 날짜 목록 API를 신규 추가한다.

선택 이유: 변경 범위가 최소이고, 기존 API 구조를 깨지 않으며, 하위 호환이 유지된다.

## API 설계

### 기존 엔드포인트 수정: `GET /api/<market>/signals`

**변경 전:** 최근 7일치 시그널을 confidence 내림차순으로 반환.

**변경 후:**
- `?date=2026-04-06` 파라미터가 있으면 해당 날짜의 시그널만 반환 (confidence 내림차순)
- 파라미터가 없으면 가��� 최근 시그널 날짜의 데이터를 반환

응답 형식은 기존과 동일:
```json
[
  {
    "id": 1,
    "symbol": "005930",
    "date": "2026-04-06",
    "signal_type": "buy",
    "price": 72400,
    "target_price": 78000,
    "stop_price": 68000,
    "reason": "상승추세 확인 + 거래량 급증",
    "confidence": 89
  }
]
```

### 신규 엔드포인트: `GET /api/<market>/signals/dates`

시그널이 존재하는 날짜 목록을 내림차순으로 반환한다. 캘린더에서 파란 점 표시 및 ◀ ▶ 네비게이션에 사용. 최근 90일 범위로 제한한다.

응답:
```json
{
  "dates": ["2026-04-06", "2026-04-03", "2026-04-02", "2026-04-01"]
}
```

### 기존 엔드포인트 수정: `GET /api/<market>/signals/summary`

**변경 전:** 가장 최근 시그널 날짜의 buy/sell/watch 카운트 ��환.

**변경 후:**
- `?date=2026-04-06` 파라미터가 있으�� 해당 날짜의 카운트 반환
- 파라미터가 ��으면 기존과 동일 (가장 최근 날짜)

## UI 설계

### 시그��� 목록 헤더

기존 "시그널 목록" 텍스트 + "전체보기" 버튼 영역에 날짜 네비게이션을 추가한다.

```
├─────────────────────────────────────┤
│  SK하이닉스         [매도]          │
│  ...                               │
└──────��────────────────────────��─────┘
```

### 캘린더 팝업

날짜 영역(예: "04.06(월)")을 클릭하면 드롭다운 캘린더가 표시된다.

- 월 단위 표시, ◀ ▶ 로 월 이동
- 시그널 있는 날짜: 흰색 텍스트 + 하단 파란 점, 클릭 가능
- 시그널 없는 날짜: 회색 텍스트, 클릭 불가
- 선택된 날짜: 파란 배경
- 오늘 날짜: 노란 테두리
- 캘린더 외부 클릭 시 닫힘

### ◀ ▶ 버튼 동작

`/api/<market>/signals/dates`에서 받은 날짜 목록을 기반으로:
- ◀: 현재 선택 날짜보다 이전의 가장 가까운 시그널 날짜로 이���
- ▶: 현재 선택 날짜보다 이후의 가장 가까운 시그��� 날짜로 이동
- 더 이상 이동할 날짜가 없으면 버튼 비활성화 (opacity 낮춤)

### Summary 카드 연동

날짜가 변경되면 `/api/<market>/signals/summary?date=선택날짜`를 호출하여 매수/��도/관찰 카운트를 갱신한다.

### localStorage 키

- `signal_date_kr`: KR 마켓 마지막 선택 날짜 (예: `"2026-04-06"`)
- `signal_date_us`: US 마켓 마지막 선택 날짜

페이지 로딩 시 해당 키가 있고 유효한 날짜(signals/dates 목록에 포함)면 사용, 아니면 가장 최근 시그널 날짜로 fallback.

## 데이터 흐름

1. 페이지 로딩 → `GET /api/<market>/signals/dates` 호출 → 가용 날짜 목록 저장
2. localStorage에서 마켓별 저장 날짜 확인 → 유효하면 사용, 아니면 dates[0] (최신)
3. 선택된 날짜로 `GET /api/<market>/signals?date=선택날짜` + `GET /api/<market>/signals/summary?date=선택날짜` ���출
4. 날짜 변경 시 → localStorage 갱신 → 시그널 목록 + Summary 카드 갱신
5. 마켓 전환 시 → 페이지 리로드 → 해당 마켓의 localStorage 날짜로 로딩
